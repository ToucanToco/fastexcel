309        """Loads a sheet lazily by index or name.
310
311        :param idx_or_name: The index (starting at 0) or the name of the sheet to load.
312        :param header_row: The index of the row containing the column labels, default index is 0.
313                           If `None`, the sheet does not have any column labels.
314                           Any rows before the `header_row` will be automatically skipped.
315        :param column_names: Overrides headers found in the document.
316                             If `column_names` is used, `header_row` will be ignored.
317        :param n_rows: Specifies how many rows should be loaded.
318                       If `None`, all rows are loaded
319        :param skip_rows: Specifies which rows should be skipped after the `header_row`.
320                          Any rows before the `header_row` are automatically skipped.
321                          It means row indices are relative to data rows, not the sheet!
322                          Can be one of:
323                          - `int`: Skip this many rows after the header row
324                          - `list[int]`: Skip specific row indices (0-based relative to data rows)
325                          - `Callable[[int], bool]`: Function that receives row index (0-based
326                          relative to data rows) and returns True to skip the row
327                          - `None`: If `header_row` is None, skips empty rows at beginning
328        :param schema_sample_rows: Specifies how many rows should be used to determine
329                                   the dtype of a column. Cannot be 0. A specific dtype can be
330                                   enforced for some or all columns through the `dtypes` parameter.
331                                   If `None`, all rows will be used.
332        :param dtype_coercion: Specifies how type coercion should behave. `coerce` (the default)
333                               will try to coerce different dtypes in a column to the same one,
334                               whereas `strict` will raise an error in case a column contains
335                               several dtypes. Note that this only applies to columns whose dtype
336                               is guessed, i.e. not specified via `dtypes`.
337        :param use_columns: Specifies the columns to use. Can either be:
338                            - `None` to select all columns
339                            - A list of strings and ints, the column names and/or indices
340                              (starting at 0)
341                            - A string, a comma separated list of Excel column letters and column
342                              ranges (e.g. `"A:E"` or `"A,C,E:F"`, which would result in
343                              `A,B,C,D,E` and `A,C,E,F`). Also supports open-ended ranges
344                              (e.g. `"B:"` to select all columns from B onwards) and from-beginning
345                              ranges (e.g. `":C"` to select columns from A to C). These can be
346                              combined for "except" patterns (e.g. `":C,E:"` to select everything
347                              except column D)
348                            - A callable, a function that takes a column and returns a boolean
349                              indicating whether the column should be used
350        :param dtypes: An optional dtype (for all columns)
351                       or dict of dtypes with keys as column indices or names.
352        """
353        return ExcelSheet(
354            self._reader.load_sheet(
355                idx_or_name=idx_or_name,
356                header_row=header_row,
357                column_names=column_names,
358                skip_rows=skip_rows,
359                n_rows=n_rows,
360                schema_sample_rows=schema_sample_rows,
361                dtype_coercion=dtype_coercion,
362                use_columns=use_columns,

364                eager=False,
365            )
366        )
